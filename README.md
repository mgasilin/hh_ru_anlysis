# Анализ спроса и предложения на рынке труда разработки и машинного обучения

Групповой проект №2 по дисциплине "Современные методы анализа данных и машинного обучения"

Команда: 46

Участники: Гасилин Максим, Солдатова Софья, Шалашов Дмитрий, Юшинский Андрей

## О проекте

В этом проекте мы решили посмотреть на рынок труда разработки и машинного обучения с двух сторон:

С одной стороны у нас есть вакансии с hh.ru, и это спрос со стороны работодателей

С другой стороны - пользователи GitHub и их репозитории, и это уже предложение со стороны потенциальных кандидатов

Наша идея в том, чтобы сопоставить вакансии и пользователей GitHub и понять, какие навыки сейчас реально нужны рынку, насколько они встречаются в открытых проектах и кто из разработчиков ближе всего к ожиданиям работодателей

## Бизнесовая часть

### Цель

Наша основная цель - собрать единый датасет по связке "вакансия - пользователь GitHub" и на его основе узнать, какие навыки и технологии чаще всего встречаются в вакансиях, насколько эти требования закрываются реальными проектами на GitHub и каких пользователей можно считать наиболее близкими к запросам работодателей

2. GitHub

Через GitHub API мы собираем пользователей и их репозитории

В выборку попадают пользователи GitHub из России, после чего мы оцениваем их по содержимому репозиториев, используемым языкам и технологиям

Дополнительно по репозиториям мы берем описание, темы, README и распределение языков

### Что получаем на выходе

На выходе у нас получается общий датасет, в котором для каждой вакансии можно найти близких пользователей GitHub по стеку, языкам, тематикам и текстовым признакам их репозиториев

На основе этого датасета мы считаем метрики соответствия вакансий стеку пользователей GitHub, анализируем востребованность технологий и строим графики по спросу и предложению

### Практическая ценность

Проект может принести практическую ценность в виде:

- анализа спроса на навыки и технологии
- оценки того, какие пользователи и какие технологии ближе всего к ожиданиям работодателей
- рекомендаций разработчику по тому, чего ему не хватает в портфолио
